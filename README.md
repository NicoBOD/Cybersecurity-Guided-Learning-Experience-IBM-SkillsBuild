# Cybersecurity Guided Learning Experience (GLE) — IBM SkillsBuild

Ce dépôt rassemble l'ensemble des livrables pédagogiques et organisationnels conçus pour les parcours de formation en cybersécurité, alignés avec la plateforme **IBM SkillsBuild**.

---

## 📂 Structure du Dépôt

Le dépôt a été réorganisé pour distinguer clairement les documents de cadrage stratégique (fichiers de travail) et les ressources pédagogiques finales destinées à l'animation des parcours.

```text
.
├── documents-de-travail/                # Fichiers internes d'organisation et de cadrage
│   ├── cahier-des-charges-gle-cybersecurite.md
│   ├── summary_gle_cyber.md             # Synthèse globale de la transition
│   ├── message-amorcage.md              # Document d'amorce initial
│   └── 00_MANIFESTE.md                  # Index détaillé d'avancement des contenus
│
├── ressources-pedagogiques/             # Contenus animés (Apprenants & Mentors)
│   ├── parcours-A-8sessions/            # Parcours court (12 heures synchrone)
│   │   ├── plans-de-seance/             # Fiches d'animation pour le mentor (A_S01 à A_S08)
│   │   ├── supports-md/                 # Supports théoriques de cours (800-1500 mots)
│   │   ├── slides/                      # Spécifications détaillées des diaporamas
│   │   ├── projet/                      # Énoncé du projet Capstone & grille d'évaluation
│   │   └── outils/                      # Banque de quiz, scripts de démo et messages types
│   │
│   └── parcours-B-20sessions/           # Parcours long (30 heures synchrone)
│       ├── plans-de-seance/             # Fiches d'animation pour le mentor (B_S01 à B_S20)
│       ├── supports-md/                 # Supports théoriques de cours (800-1500 mots)
│       ├── projet/                      # Sujets (Capstone B, mini-projets) et grille
│       └── outils/                      # Banque de quiz, scripts de démo et messages types
└── README.md                            # Ce guide d'accueil
```

## 🛠️ Propositions de Présentation & de Valorisation

Afin d'offrir la meilleure expérience utilisateur possible pour les mentors et les apprenants qui consulteront ce dépôt, voici 3 propositions concrètes. **Merci de me faire savoir celle que vous préférez pour que je l'implémente :**

### Option 1 : Site de documentation interactif (Docsify / MkDocs Material) — *Recommandé*
*   **Fonctionnement** : Déploiement automatique via **GitHub Pages** d'un site web à partir de ces fichiers Markdown, sans aucune compilation requise.
*   **Avantages** :
    *   Interface web fluide, moderne et épurée (style "documentation officielle").
    *   Barre de recherche textuelle globale (permet aux apprenants de chercher un concept précis comme "CIA" ou "VLAN").
    *   Menu latéral interactif avec les sessions A01 à A08 et B01 à B20.
    *   Lecture idéale sur mobile et tablette pour les apprenants en classe.

### Option 2 : Compilation automatique des Supports en PDF (GitHub Actions CI/CD)
*   **Fonctionnement** : Un workflow GitHub Actions se déclenche à chaque modification des supports. Il utilise un moteur comme `pandoc` ou `marp` pour compiler automatiquement chaque support `.md` en PDF de haute qualité et les dépose dans les *Releases* du dépôt.
*   **Avantages** :
    *   Les mentors et apprenants peuvent télécharger les supports sous format PDF imprimable ou liseuse en un clic.
    *   Mise en page propre et chartée de manière automatisée.

### Option 3 : Kanban de suivi de promotion (GitHub Projects)
*   **Fonctionnement** : Configuration d'un template de projet GitHub sous forme de tableau Kanban pré-rempli avec les jalons de la formation.
*   **Avantages** :
    *   Permet au mentor de glisser les sessions de "À faire" à "En cours" puis "Terminé" pour structurer visuellement l'avancée de la promotion.
    *   Idéal pour lier les livrables des apprenants (les mini-projets) directement aux cartes du projet.
